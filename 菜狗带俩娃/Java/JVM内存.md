由大到小：jdk（工具包）          jre（编译环境）       jvm（Java虚拟机）



jvm中**主要**有三块内存空间：堆、栈、方法区

方法区：类加载区classloader，将硬盘上的.class字节码文件装载到jvm时，会将字节码文件存放到方法区当中，也就是说方法区中存储的时代码片段。因为类需要加载，所以方法区中最先有数据，静态变量也在方法区

栈：存储方法和方法调用时用到的局部变量。在方法被调用的时候，该方法需要的内存空间在栈中分配，不调用方法时不分配内存，方法调用时压栈，释放时弹栈

数据结构通常时存放数据的容器，而该容器可能存在不同的结构，常见数据结构：数组，链表，图，树，栈，队列......

栈数据结构：stack，后进先出

堆：凡是通过new运算符创建的对象，都存储在堆内存当中，new运算符的作用就是在堆内存中开辟一块空间。

Java中的垃圾回收器GC主要针对回收的是堆内存当中的垃圾数据，在没有任何引用指向该对象的时候，该数据就会被垃圾回收器回收

当**“空引用**”访问**实例******对象****相关的数据时，就会出现空指针异常-->即类对象可以看作是一个指针，当该类对象有实际的内存地址时，才可以访问位于堆内存中的成员变量，当类对象地址为空时，就无法访问堆内存中的成员变量。访问静态变量时不会出现空指针异常
