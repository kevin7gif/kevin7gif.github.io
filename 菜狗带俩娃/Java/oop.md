方法重载和方法覆盖的区别

方法重载发生在同一个类当中

方法覆盖发生在具有继承关系的父子类之间

方法重载是一个类中，方法名相同，参数列表不同

方法覆盖是具有继承关系的父子类，并且重写之后的方法必须和之前的方法一致（方法名一致，参数列表一致，返回值类型一致）





### **方法覆盖和多态联系起来才有意义**

静态方法不存在方法覆盖，因为静态方法和对象无关，直接通过类名. 来调用

私有方法和构造方法不能被继承

方法覆盖时的返回值：

若为基本数据类型，则覆盖前后的返回值类型要一致

若为引用数据类型，则可以由大变小，不能由小变大，即覆盖前返回值类型范围大，覆盖之后返回值类型范围小时可以通过的。反之则不行

**上转型与下转型（多态）**

**上转型**与**下转型**的**前提**是两个类要有**继承**关系

```java
//向上转型
Animal a2 = new Cat();

//假设move()方法是由父类继承而后重写的方法
a2.move();//可以实现

//假设catchMouse()方法是子类Cat类独有的方法
a2.catchMouse();//会报错
```

错误原因：

编译器只知道a2的类型是Animal，去Animal.class文件中找catchMouse方法，结果没有找到，所以静态绑定失败，编译报错

如果想调用catchMouse方法，则要使用“向下转型”

```java
Animal a2 = new Cat();
Cat x=(Cat)a2;
a2.catchMouse();
```





```java
//假设Animal是父类，Bird类和Cat类都继承了Animal类
Animal a6 = new Bird();//表面上a6是一个Animal，运行时实际上是一个Bird
Cat y = (Cat) a6;
y.catchMouse();

上述代码编译时没问题，运行时有问题，显示ClassCastException异常
//编译器检测到a6这个引用是Animal类型，而Animal和Cat之间存在继承关系，所以可以向下转型，编译没问题
 
//运行阶段，堆内存实际创建的对象是Bird对象，在实际运行过程中，拿着Bird对象转换成Cat对象就不行了，因为Bird和Cat之间没有继承关系


```



Java程序分为编译阶段和运行阶段

编译阶段：对于编译器来说，编译器只知道a2的类型是Animal，所以编译器在检查语法的时候，会去Animal.class字节码文件中查找move()方法，找到了，绑定上move()方法，编译通过，静态绑定成功。（编译阶段属于静态绑定）

运行阶段：实际上在堆内存中创建的Java对象是Cat对象，所以执行move方法的时候，真正参与move的对象是猫，所以运行阶段会动态执行Cat对象的move()方法，这个过程属于运行阶段绑定（运行阶段属于动态绑定）

**instanceof** 关键字

作用：避免向下转型时出现ClassCastException异常

可以在运行阶段动态判断引用指向的对象的类型，instanceof运算结果为true或false

假设：c是一个引用，c变量保存了内存地址指向了堆中的对象

(c instanceof Cat)为true表示：c引用指向的堆内存中的Java对象是一个Cat

(c instanceof Cat)为false表示：c引用指向的堆内存中的Java对象不是一个Cat

例如在上面下转型的代码中：

```java
Animal a6 = new Bird();
if(a6 instanceof Cat){
    Cat y = (Cat) a6;
	y.catchMouse();
}
//这样即可避免出现ClassCastException错误
```

**super关键字**

用来在子类中访问从父类继承下来的特征，当子类和父类有同名属性时，或者子类重写了父类的方法时，super不能省略



super()：用来在子类的构造方法中初始化由父类继承下来的属性，**并且super()只能出现在构造方法的第一行**

super.  ：用来访问由父类继承下来的属性或者重写的方法



this和super不能使用在静态方法中



当子类从父类继承了一个name属性并且子类自己也定义了一个name属性时，相当于现在有两个name属性（Java允许子类中出现和父类一样的同名变量/属性）

this.name：当前对象的name属性

super.name：当前对象的父类型特征中的name属性，或者访问由父类继承下来的并且在子类中重写了的方法

**final关键字**

final修饰的类无法继承

final修饰的方法无法覆盖

final修饰的引用一旦指向某个对象，则不能再重新指向其他对象，但是该引用指向的对象内部的数据是可以修改的

final修饰的实例变量，系统不负责赋默认值，要求程序员手动赋值

static final 联合修饰的变量称为'"常量"'，常量名建议全部大写，每个单词之间用下划线连接

常量和静态变量的区别：都存储在方法区，并且都是在类加载时初始化，常量的值不能改变

**抽象类**

在class前加上abstract关键字，该类就为抽象类

抽象类无法实例化，无法创建对象，所以抽象类是用来被子类继承的

final和abstract不能联合使用，这两个关键字是对立的

抽象类的子类可以是抽象类也可以是非抽象类

抽象类虽然无法实例化，但是有构造方法，这个构造方法是供子类使用的

抽象类中不一定要有抽象方法，但是抽象方法必须出现在抽象类中

一个非抽象的类，继承抽象类，必须将抽象类中的抽象方法进行覆盖

**接口**

接口是一种“引用数据类型”，编译之后也是一个class字节码文件

接口是完全抽象的，抽象类是半抽象的

定义：[修饰符列表]  interface  接口名  {    }

接口支持多继承

接口中只有常量和抽象方法（不能有方法体），都是用public修饰的

接口中抽象方法的public abstract可以省略

接口中常量的public static final可以省略

当一个非抽象类实现接口的话，必须将接口中所有的抽象方法全部实现（重写、覆盖）

一个类可以实现多个接口

继承和接口同时出现时，继承（extends）在前，接口（implements）在后

使用接口时，可以使用多态（父类型的引用指向子类型对象）



接口在开发中的作用

类似于多态（面向抽象编程，降低程序耦合度，提高程序扩展力）
