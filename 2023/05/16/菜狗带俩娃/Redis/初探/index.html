NoSQL四大分类

KV键值对：

  • 新浪：redis
  • 美团：redis+tair
  • 阿里、百度：redis+memecache

文档型数据库（bson格式和json一样）：

  • MongoDB
    • 基于分布式文件存储的数据库，C++编写，主要用来处理大量 的文档
    • 介于关系型数据库和非关系型数据库中间的产品，是非关系型数据库中功能最丰富，最像关系型数据库的
  • ConthDB

列存储数据库：

  • HBase
  • 分布式文件系统

图关系数据库：

  • 用来存储关系的

Redis（非关系型数据库）

是完全开源免费的，使用C语言开发的高性能键值对(key-value)数据库

Redis是一个简单的，高效的，分布式的，基于内存的缓存工具

Redis特点

  • 性能极高
    • 数据是存储在内存中的（读的速度是110000次/s，写的速度是81000次/s）
  • 丰富的数据类型
    • string，Hash，Set，List以及Ordered Set等
  • 原子性
    • 所有操作都是原子性的，要么成功执行，要么全不执行
  • 丰富的特性
    • 支持publich/ subscribe，通知，key过期等特性
  • 高速读写
    • 使用自己实现的分离器，代码量很短，没有使用lock(MySQL)，效率很高
  • 持久化
    • redis直接将数据存储到内存中，要将数据保存到磁盘上，可以用两种方式来实现持久化过程。
    • 定时快照：每隔一段时间将全部数据写到磁盘，代价非常高
    • 基于语句追加：只追踪变化的数据，但是追加的og可能过大，同时所有操作均重新执行一遍，回复速度慢

对于内存系统来说，没有上下文切换效率是最高的！

Redis缺点

  • 耗内存
    • 占用内存过高
  • 在线扩容
    • redis难以支持在线扩容，尤其是在集群场景里，当存储容量达到上限后，在线扩容会非常困难
  • redis的瓶颈是根据机器的内存和网络带宽，使用单线程
  • 核心：redis是将所有的数据全部放到内存中的，所以说使用单线程去操作效率高，多线程（CPU上下文切换：耗时的操作！！！）

Redis应用场景

可以用在数据库，缓存，秒杀，计数器，排行榜，热点数据（经常会被查询但不经常被修改或删除的数据），分布式锁，分布式ID，消息中间件等大部分功能与性能密切相关的场景里
